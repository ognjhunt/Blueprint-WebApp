# Blueprint Beta — Tester Onboarding

Welcome to the Blueprint beta. This folder is the tester-facing "what to expect / how it
works / how to get help" set for the ~100-person external beta cohort. It is written to be
honest about what the beta does and does not do today.

Blueprint is **capture-first**: real-site capture supply is the product entry point, and the
sellable outputs are **Task Evaluation Runs** and **Post-Training Data Packages** built on
those captures. World models, simulation, and generated media are *support* substrates inside
those runs — they are not the product and are not claimed as physical-readiness proof.

## What this beta honestly is

- A **scoped, sim/review-grade** beta — Task Evaluation Runs and Post-Training Data Packages
  on real captures, **industrial-first** (warehouse / factory / fulfillment) plus other allowed
  location types.
- **Not** a live-robot certification, a physical-readiness guarantee, or a world-model-fidelity
  claim. Every quantitative result in the beta is **advisory** until owner-system proof exists.

## One support channel

For anything — access blockers, capture problems, questions — email **hello@tryblueprint.io**
or use the in-app **Support** page. Include the exact page/package/request URL, your
organization, expected outcome, and the actual blocker. Support is prioritized by severity
(access blockers and live review paths first).
