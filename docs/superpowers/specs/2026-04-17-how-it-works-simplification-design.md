# How It Works Simplification Design

Date: 2026-04-17

Owner: `webapp-codex`

Status: Approved for implementation by user-directed autonomy

## Goal

Simplify `/how-it-works` into a shorter proof-led page that explains Blueprint's exact-site training loop in a way a buyer can understand quickly.

## Problem Summary

The current page is conceptually strong but too expansive:

- large hero with research framing
- variant-specific step section
- proof chain
- comparison
- common jobs
- proof stories
- closing CTA

This creates a long conceptual walk when the page should explain one idea clearly:

- start with one real site
- branch realistic variation around it
- run, review, and export from the same grounded environment

## Recommended Direction

Rebuild the page into five sections:

1. Hero
2. Exact-site loop
3. Proof path
4. Where Blueprint fits
5. Closing CTA

The page should become more visual and less essay-like.

## New Information Architecture

### 1. Hero

Headline:
- `Start with one real site. Train around it.`

Support line:
- `Blueprint anchors training and evaluation to one exact facility, then lets a team branch realistic variation without losing the real-site starting point.`

### 2. Exact-Site Loop

Purpose:
- show the loop visually and quickly

Keep one simplified loop:
- anchor
- vary
- run and compare
- export

### 3. Proof Path

Purpose:
- show how the buyer path moves from site evidence into hosted review and exports

Keep proof visuals, but compress the supporting copy.

### 4. Where Blueprint Fits

Purpose:
- retain the generic simulation vs exact site vs exact site plus controlled variation comparison

Trim the explanation and avoid additional use-case or case-study sections.

### 5. Closing CTA

Purpose:
- send the user into world models or sample deliverables

## Acceptance Criteria

- the page is materially shorter
- the exact-site loop is understandable within seconds
- case studies and extra use-case grids are removed
- proof and comparison remain visible
