# About, FAQ, and Contact Simplification Design

Date: 2026-04-17

Owner: `webapp-codex`

Status: Approved for implementation by user-directed autonomy

## Goal

Simplify `/about`, `/faq`, and `/contact` into shorter, clearer public pages that match the new premium editorial direction already established on Home, World Models, Hosted Review, Pricing, Governance, and How It Works.

## Shared Rules

- fewer sections
- faster scan
- fewer repeated explanatory blocks
- exact-site, capture-first, provenance-safe language preserved
- no generic AI marketplace tone

## Page Directions

### About

Turn About into a concise company-framing page:

1. short hero
2. founder / operating posture card
3. what Blueprint is / is not
4. one short deployment-decision story
5. closing CTA

Remove repeated grids that explain the same company posture three different ways.

### FAQ

Turn FAQ into a shorter objections page:

1. short hero
2. one section of the highest-signal questions only
3. one closing CTA block

Keep only the questions that materially decide fit:
- what is the product
- what does a buyer receive
- what is hosted evaluation
- exact-site vs adjacent-site proof
- deployment guarantee boundary
- what if the site is not in the catalog
- response speed
- booking path

### Contact

Turn Contact into a cleaner intake page:

1. tighter hero
2. form as the main action surface
3. one response / cadence card
4. one fast-path card
5. preserve hosted mode and city-specific guidance

Remove excess decorative and repeated support chrome while keeping the actual functional routing intact.

## Acceptance Criteria

- each page is materially shorter
- each page reaches its primary action or explanation faster
- Contact still preserves hosted-mode, city, and persona behavior
- FAQ no longer feels like a long internal glossary
- About reads like a premium company page, not stacked internal notes
