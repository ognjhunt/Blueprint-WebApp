# Robot Team Growth Agent - Current Focus

## Priority

Maintain the generic buyer motion for robot teams that need exact-site world models, site packages, and hosted evaluation built from real capture.

This cycle should keep the reusable system tighter than any single city plan:

1. lock the baseline ICP and message hierarchy for robot-team buyers
2. keep the proof pack anchored to product surfaces that exist now
3. define the buyer funnel and handoff points that downstream agents should inherit
4. surface message or ops drift that weakens truthful buyer understanding

## Product Truth To Preserve

- Blueprint is capture-first and world-model-product-first.
- The primary sellable outputs are site-specific world models, site packages, and hosted sessions on exact sites.
- Hosted evaluation is a managed path on one exact site, not a blanket deployment guarantee.
- Rights, privacy, provenance, and commercialization controls remain part of the product record.
- Qualification and readiness can support the sale, but they must not become the main story.

## Reusable Buyer System In Scope

- ICP definitions for robot-team buyers
- message hierarchy across buyer-facing surfaces
- proof-pack requirements
- hosted-session demo framing
- buyer-funnel stage definitions and handoffs

## Grounded Product Surfaces Available Now

- `/world-models` catalog and listing flow
- `/proof` proof hub plus proof reel assets
- sample deliverables and package framing
- hosted-session setup and workspace routes
- buyer request capture through the contact and hosted-evaluation paths

Keep any recommendation bounded by those current surfaces unless the recommendation is clearly marked as new downstream work.
