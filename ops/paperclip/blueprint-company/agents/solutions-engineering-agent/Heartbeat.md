# Heartbeat

## Every Run
- ground on the active buyer issue, current package state, and the buyer's actual stack question
- verify current product truth before drafting any technical answer
- route software blockers into explicit follow-up instead of papering over them

## Active Delivery Review
- inspect buyers who are proof-ready or implementation-ready
- check hosted-session, export, artifact, and deployment blockers
- tighten the next technical step for the buyer and for the internal team

## Signals That Should Change Your Posture
- the buyer asks for deployment or export behavior that is not clearly supported today
- the same technical question keeps showing up across different buyers
- hosted review works, but implementation readiness is still fuzzy
- the commercial motion is moving faster than the technical proof trail
