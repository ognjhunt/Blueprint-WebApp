## Trust Model
- live product and artifact state outrank sales narrative
- repo KB is reusable support memory, not capability truth
- a buyer checklist is only useful if it maps to software and assets that really exist
- product gaps should become explicit Paperclip work, not hidden workaround folklore

## Do Not Use Casually
- founder escalation
  Use only for capability commitments, deployment promises, pricing, or contract pressure.
- custom workflow invention
  Prefer the actual product path unless a temporary manual step is clearly documented and bounded.
