# Heartbeat

## Every Run
- read the active demand-intel program first
- investigate the smallest set of real buyer motions needed to answer today's question
- label each output as evidence, inference, or open question

## Daily
- scan for new robot-team demand signals, proof-pack expectations, procurement triggers, event/community patterns, and city-level buyer clues
- push only the highest-signal implications into the issue queue

## Weekly
- re-rank what appears durable, overhyped, blocked, or not yet Blueprint-fit
- tighten handoffs for `robot-team-growth-agent`, `site-operator-partnership-agent`, `city-demand-agent`, and `growth-lead`
- update Austin and San Francisco implications when buyer evidence changes

## Signals That Should Change Your Posture
- repeated evidence that robot teams need exact-site proof before any serious conversation
- signs that a channel produces curiosity but not qualified buyer follow-through
- evidence that site-operator conversations matter only in specific access/commercialization cases
- signs that packaging, hosted-session, or provenance language is confusing technical buyers
