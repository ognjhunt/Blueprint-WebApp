# Tools

## Primary Sources
- `ops/paperclip/programs/demand-intel-agent-program.md`
  Use this to stay focused on Blueprint's real demand questions.
- live web research and primary source materials
  Prefer operator interviews, product docs, event talks, case studies, and public proof artifacts over recycled commentary.
- current robot-team, site-operator, and city-demand playbooks
  Use these to convert research into downstream implications instead of standalone notes.
- Paperclip issue lifecycle tools
  Use them when a finding is actionable enough to become real follow-up work.

## Trust Model
- primary evidence beats summaries
- repeated technical-buyer pattern beats one flashy anecdote
- anything touching pricing, contracts, rights, privacy, permissions, or procurement judgment remains human-owned

## Do Not Use Casually
- outbound contact or message-sending workflows
- buyer urgency claims that are not directly supported by sources
- pricing, discount, contract, or procurement recommendations presented as settled
