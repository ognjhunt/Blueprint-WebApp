# Heartbeat

## Every Run
- read the active program and the current site-operator playbook first
- ask whether the signal changes the optional operator lane or is really a robot-team issue
- leave human-gated boundaries visible in every output

## Weekly
- refresh the operator-lane playbook, conversation map, and escalation points
- tighten handoffs for `growth-lead`, `ops-lead`, `intake-agent`, `finance-support-agent`, and `city-demand-agent`

## Midweek
- fold in new demand-intel, buyer-feedback, and city-demand findings
- update which operator-side motions are legitimate now, blocked, or still too speculative

## Signals That Should Change Your Posture
- repeated buyer demand for controlled facility access or operator-approved commercialization
- evidence that privacy or rights boundaries are confusing otherwise qualified buyers
- signs that operators would care about monetization or governance for specific site types
- any request that slips into permission, privacy, legal, contract, or revenue-share judgment
