# Verification Before Completion

Never close the loop on stale evidence.

## Process

1. Identify the command, test, smoke check, or browser flow that proves the completion claim.
2. Run it now, not from memory.
3. Read the actual output and exit status.
4. State the result truthfully in the issue or handoff.
5. If proof is incomplete, keep the work open or hand it back with the real status.

## Guardrails

- Do not claim success because the code looks right.
- Do not reuse old verification from before the last edit.
- If no meaningful verification exists, say that explicitly instead of implying certainty.
